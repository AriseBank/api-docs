Simple, beautiful, responsive documentation.

api-docs is a minimalist toolkit for producing great documentation. The intent of this toolkit is to make larger documentation projects more approachable, like the sort that are needed for documenting cross-platform and web service APIs.

There's no database here, no code generation, no new syntaxes to learn. You'll compose your docs using HTML, optionally making use of some special tags we've design just for marking up source code.

The documentation you're reading now was built using api-docs, so if you like what you see, you're half way there.

Thank you.

api-docs wouldn't look so amazing were it not for the example set by the team at Stripe. A quick look at their API reference documentation and you'll be able to see why. It was my experience working with their product and their docs that first inspired me to look for a platform on which to write my own beautiful docs, leading eventually to building api-docs.

Getting Started

To start a new documentation project with api-docs, the first thing you'll want to do is clone the GitHub project. For this you'll need to use Git.

If you're unfamiliar with Git, api-docs might not be the tool you were hoping for. If you are in need, here are some great resources for learning Git.

Despite being minimal, api-docs still relies on a few dependencies, namely NPM, bower, and Grunt, and you'll need to install them before you can really get rolling.

The last and most important thing you'll need is a good text editor. Personally, I love Sublime Text, but I hear great things about Atom, too.

Setting up your documentation project

To create your documentation project, start by cloning the api-docs project from GitHub, then install its few dependencies by running npm install and bower install.

$ git clone git@github.com:gobeebo/api-docs.git \
    your-project-name
 
your-project-name $ npm install && bower install

Building

You'll do all of your writing using HTML, with each major section of your documentation living in a dedicated file. api-docs processes and decorates your files to create a single doc file, index.html, that will live in the root of your project (unless you tell it to go somewhere else).

The building of your doc file is handled by Grunt, which you can run anytime with the command grunt dist. You can also have Grunt building continuously in the background, rebuilding your doc file in response to changes you make, by running the command grunt watch.

Building your documentation

Build your doc file in the background using with the command grunt watch, or on demand using grunt dist.
